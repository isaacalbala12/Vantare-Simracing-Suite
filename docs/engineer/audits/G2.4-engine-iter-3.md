## Iteración 3 — Cambios aplicados

1. **60-second moving average via ring buffer**:
   - Nuevo tipo `tempSample` con `timeMS`, `waterTemp`, `oilTemp`.
   - Slice `samples []tempSample` en Monitor.
   - `addAndPruneSamples()` añade muestra y elimina entradas > 60s.
   - `avgWaterTemp()` / `avgOilTemp()` retornan promedio si ≥ 10 muestras.
   - Tests: `TestMonitor_WaterAllClear_FiresWhenAvgBelowReArm`, `TestMonitor_OilAllClear_FiresWhenAvgBelowReArm`, `TestMonitor_AllClearRequiresMinSamples`, `TestMonitor_AllClear_ReArmWhenTempRises`.

2. **`EventWaterTempAllClear` (priority 5)**:
   - Se dispara cuando el promedio de agua baja de `waterHighReArm` (95°C).
   - Re-arm cuando promedio ≥ `waterHighThreshold` (105°C).

3. **`EventOilTempAllClear` (priority 5)**:
   - Se dispara cuando el promedio de aceite baja de `oilHighReArm` (120°C).
   - Re-arm cuando promedio ≥ `oilHighThreshold` (130°C).

4. **`EventEngineStalled`**:
   - Gate: `curr.Player.Speed < 5`.
   - Cooldown 2 minutos (120s), con zero sentinel para primer disparo.
   - Tests: `TestMonitor_EngineStalled_FiresWhenSpeedBelow5`, `TestMonitor_NoStalledAtHighSpeed`, `TestMonitor_StalledCooldown`.

5. **Session phase gate**:
   - Solo permite eventos cuando `frame.Session.GamePhase` es Green(5), FullCourseYellow(6), o Checkered(8).
   - `sessionPhaseOK()` helper function.
   - Tests: `TestMonitor_PhaseGate_BlocksNonGreen`, `TestMonitor_PhaseGate_AllowsGreen`, `TestMonitor_PhaseGate_AllowsFCY`, `TestMonitor_PhaseGate_AllowsCheckered`.

6. **Tests existentes actualizados**:
   - `mkFrame` ahora incluye `Speed: 50` y `GamePhase: 5` para evitar falsos stalled.
   - Nuevo helper `mkFrameSpeed` para tests de velocidad/fase.

## Re-auditoría post-iter-3

**Diferencias materiales que PERSISTEN:**

1. **Hot oil AND water combinado** — CC previene duplicación con `folderHotOilAndWater` cuando ambos están calientes. Vantare dispara 2 mensajes separados. Diferido a iter-4.

2. **Oil pressure / Fuel pressure warnings** — Requieren leer `EngineOilPressureWarning` y `EngineFuelPressureWarning` flags. LMU no expone estos como flags separados. `PARITY_BLOCKED` — requiere Extended buffer.

3. **Per-class thresholds via `carClassData.json`** — CC permite configurar `maxSafeWaterTemp` y `maxSafeOilTemp` por clase. Vantare usa valores fijos. Aceptado para alpha 3.

4. **`SessionRunningTime` gate** — CC requiere `SessionRunningTime > 60 * MinutesIntoSession(1)` para temp monitoring, y `> 30s` para pressure checks. Vantare no implementa time gate. Diferido a iter-4 si es necesario.

**Diferencias NO materiales (aceptadas):**
- Vantare usa `PriorityNormal` (10) para todos los mensajes, CC usa 5 para all-clear y 3 para stalled. Mantenemos uniforme por simplicidad.
- CC usa `EngineData` inner class con getEngineStatusFromAverage/getEngineStatusFromCurrent; Vantare usa implementación más directa con slice y helpers.
