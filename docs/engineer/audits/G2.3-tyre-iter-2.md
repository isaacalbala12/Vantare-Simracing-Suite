## Re-auditoría post-iteración 2

**Diferencias materiales que PERSISTEN:**

1. **`lapsIntoSessionBeforeTempMessage = 2`** — CC espera 2 vueltas antes de
   alertar de temperatura. Vantare alerta desde vuelta 1. Decisión:
   implementar en iter-3.

2. **Tyre type detection** — CC identifica Soft/Medium/Hard/Intermediate/Wet
   y ajusta thresholds por compound. Vantare no distingue type. Decisión:
   `PARITY_BLOCKED` — requiere leer `TyreType` del mmap LMU (offset
   desconocido, pendiente live capture).

3. **Inner/Middle/Outer temperatures** — CC analiza 3 puntos por rueda para
   camber analysis. Vantare lee 1 temperatura por rueda. Decisión:
   `PARITY_BLOCKED` — requiere decodificar `LMUWheel` struct (260 bytes × 4),
   offsets no confirmados.

4. **Brake temp monitoring per-axis** — CC distingue front/rear brakes.
   Vantare tiene 4 brake temps pero solo dispara warnings genéricos.
   Decisión: implementar brake temp monitor separado en iter-3 (con
   caveat de u8 truncation).

5. **Wear sub-messages** — CC tiene carpetas separadas para
   `knackered_*` (>75%), `worn_*` (15-30%), `minor_*` (5-15%),
   `scrubbed_*` (<5%). Vantare solo dispara "wear_high" (>75%).
   Decisión: añadir wear_minor (20-50%) en iter-3.

6. **Flat spot detection / Wheel lock / Wheel spin** — Features CC no
   implementadas. Decisión: G3+ scope (requieren análisis per-lap).

7. **Per-wheel granularity** — Vantare ya dispara por rueda individual
   (FL/FR/RL/RR) en payload. **MATCH_CONFIRMED** en iter-2.

**Diferencias NO materiales (aceptadas):**

- CC usa priority 5 para tyre messages, Vantare usa 10. Mantenemos
  PriorityNormal uniforme.

## Plan para iter-3

1. Implementar `lapsIntoSessionBeforeTempMessage = 2`:
   - Trackear `lapNumber` del frame
   - Suprimir temp/wear events si `lapNumber < 2`
2. Añadir `EventTyreWearMinor` para 20-50% wear (preaviso antes de
   worn-out). Cooldown más largo (2 min) para no spamear.
3. Implementar brake temp monitor básico (front/rear) con threshold
   CC: COLD<200, WARM<600, HOT<800, COOKING≥800. Caveat u8 → max 255
   implica que HOT/COOKING no son distinguibles en LMU.
4. Re-auditoría final → `PARITY_OK` o `PARITY_PARTIAL`.

  (no confirmado en captura).
- Inner/middle/outer temps: requiere decodificar `LMUWheel` (260 bytes × 4).
