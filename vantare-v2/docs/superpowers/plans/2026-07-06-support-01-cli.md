# SUPPORT-01 — CLI de soporte Go local (no distribuido)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Construir un CLI de soporte local en Go que permita hacer lookup de usuarios, otorgar/revocar entitlements manualmente y forzar reset de device, sin exponer service-role en la app de escritorio.

**Architecture:** Un binario Go independiente (`cmd/vantare-admin/main.go`) que usa la `SUPABASE_URL` y `SUPABASE_SERVICE_ROLE_KEY` para ejecutar queries de lectura (REST) y mutaciones (SQL directo vía client o RPCs). Requiere que `SQL-01` esté aplicado (tablas + RPCs). No depende de Wails ni del frontend; compila aparte.

**Tech Stack:** Go stdlib (`net/http`, `encoding/json`, `os`, `fmt`), flags de CLI manuales (`os.Args`), Supabase REST API con service-role. Cero dependencias nuevas.

---

## Contexto

- Las tablas (`profiles`, `user_entitlements`, `devices`, `license_events`, `stripe_customers`, `stripe_subscriptions`) y los RPCs (`get_account_entitlements`, `reset_active_device`) son creadas por `SQL-01`.
- El service-role **nunca debe ir en la app de escritorio**; este CLI se ejecuta solo desde la máquina del equipo (Isaac).
- No hay UI ni panel web (decisión documentada en `docs/stripe-licensing-status-audit.md` §6).

package main

import (
	"fmt"
	"os"
	"strings"
)

func main() {
	cmd, args := parseArgs(os.Args)
	if cmd == "" {
		fmt.Fprintln(os.Stderr, "uso: vantare-admin <lookup|grant|revoke|device-reset|events> [args...]")
		os.Exit(1)
	}
	if err := validateEnv(); err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}
	switch cmd {
	case "lookup":
		handleLookup(args)
	case "grant":
		handleGrant(args)
	case "revoke":
		handleRevoke(args)
	case "device-reset":
		handleDeviceReset(args)
	case "events":
		handleEvents(args)
	default:
		fmt.Fprintf(os.Stderr, "comando desconocido: %s\n", cmd)
		os.Exit(1)
	}
}

func parseArgs(args []string) (cmd string, rest []string) {
	if len(args) < 2 {
		return "", nil
	}
	return args[1], args[2:]
}

func validateEnv() error {
	if os.Getenv("SUPABASE_URL") == "" || os.Getenv("SUPABASE_SERVICE_ROLE_KEY") == "" {
		return fmt.Errorf("faltan SUPABASE_URL y/o SUPABASE_SERVICE_ROLE_KEY")
	}
	return nil
}

func handleLookup(args []string) {
	if len(args) < 1 {
		fmt.Fprintln(os.Stderr, "uso: vantare-admin lookup <email|user_id>")
		os.Exit(1)
	}
	query := args[0]
	baseURL := os.Getenv("SUPABASE_URL")
	key := os.Getenv("SUPABASE_SERVICE_ROLE_KEY")

	// Buscar perfil por email o id
	profile, err := fetchProfile(baseURL, key, query)
	if err != nil {
		fmt.Fprintf(os.Stderr, "error: %v\n", err)
		os.Exit(1)
	}
	// Buscar entitlements
	entitlements, err := fetchEntitlements(baseURL, key, profile.ID)
	if err != nil {
		fmt.Fprintf(os.Stderr, "error: %v\n", err)
		os.Exit(1)
	}

	fmt.Printf("ID:       %s\n", profile.ID)
	fmt.Printf("Email:    %s\n", profile.Email)
	fmt.Printf("Plan:     %s\n", profile.Plan)
	for _, e := range entitlements {
		fmt.Printf("  %s (%s)\n", e.ProductKey, e.Status)
	}
}

- [ ] **Step 5: Implementar grant, revoke, device-reset, events**

Mismo patrón RED→GREEN para cada comando:

- **grant `<user>` `<product_key>`**: POST a `/rest/v1/user_entitlements` con `{user_id, product_key, status: "active"}` (upsert con `ON CONFLICT (user_id, product_key) DO UPDATE SET status = 'active'`). O usar RPC `grant_entitlement` si se añade a `SQL-01`.
- **revoke `<user>` `<product_key>`**: PATCH o DELETE lógico (status `expired`).
- **device-reset `<user>`**: llama al RPC `reset_active_device` con la service-role key NO funciona porque el RPC usa `auth.uid()` (vacío con service-role). Alternativa: SQL directo que limpia la fila de dispositivo del usuario, INCLUYENDO `last_reset_at` para no dejar el rate-limit bloqueado:
  ```sql
  UPDATE devices SET fingerprint_hash = NULL, last_reset_at = NULL WHERE user_id = $1;
  ```
  (Corrección de consistencia con SQL-01: la tabla `devices` ya no usa `reset_count_24h`, usa `last_reset_at`.)
- **events `<user>`**: GET a `/rest/v1/license_events?user_id=eq.<uuid>&order=created_at.desc&limit=20`.

Cada comando usa `validateEnv()` + flags de ayuda.
