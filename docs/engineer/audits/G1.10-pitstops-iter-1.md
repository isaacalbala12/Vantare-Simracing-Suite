### Gates — limitador

**Engage limiter:**
- `!previousGameState.PitData.InPitlane && currentGameState.PitData.InPitlane`
- `currentGameState.PitData.limiterStatus == INACTIVE`
- `CarSpeed > 1`
- `PitSpeedLimit == -1.0f || pitlaneHasSpeedLimit()`
- Cooldown: `timeOfLastLimiterWarning + 30s`

**Disengage limiter:**
- `SectorNumber == 1` (o iRacing: OnOutLap && !InPitlane && !IsApproachingPitlane)
- `!currentGameState.PitData.InPitlane`
- `limiterStatus == ACTIVE`
- `Now > timeOfDisengageCheck` (2s después de salir)
- Cooldown: `timeOfLastLimiterWarning + 30s`

### Gates — Watch your speed

- Entrada a pits sin limiter disponible
- `PitSpeedLimit == -1.0f || pitlaneHasSpeedLimit()`
- `SessionRunningTime > 30`
- Cooldown: `timeSpeedInPitsWarning + 120s`

## Non-material differences (accepted)

- Vantare usa `PlayerTelemetry.InPits` de LMU (offset 198). CC usa `PitData.InPitlane` que puede ser más rico.
- Vantare no tiene `PitData.LimiterStatus` — se infiere de `InPits` + flags de game state para limiter.
- Vantare no tiene `PitData.HasMandatoryPitStop` — LMU no expone esto directamente.
