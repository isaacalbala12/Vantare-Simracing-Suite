# E2E-01 — Flujo real de pago en test mode

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Verificar el flujo completo login→checkout→webhook→entitlement en Stripe test mode, demostrando que la beta pública puede cobrar y entregar licencias reales.

**Architecture:** Prueba manual guiada (smoke test documentado) que cubre el happy path y 2 escenarios de error. No añade tests automatizados nuevos (los existentes de `LoginScreen.test.tsx`, `PaywallScreen.test.tsx`, `HubApp.test.tsx` y `access-policy.test.ts` ya cubren los estados por separado). El valor es probar la integración real con servicios externos.

**Tech Stack:** App Vantare (build empaquetada Wails), navegador externo, Stripe test mode, Supabase Studio.

### Task 2: Ejecutar flujo happy path

- [ ] **Step 1: Login como usuario nuevo**

1. Abrir `bin/vantare.exe`.
2. En LoginScreen, modo "Crear cuenta": email `e2e-test@vantare.dev`, password `Test123!`.
3. Si email confirmation está activo, confirmar desde el email de Supabase.
4. Iniciar sesión con email/password.
5. Expected: entra al Hub con banner LicenseBanner Free. Dashboard muestra cards de Free. Topbar muestra secciones Pro bloqueadas (disabled).

- [ ] **Step 2: Ir a Paywall y comprar Overlays**

1. Ir a Ajustes → Cuenta. Plan: "Free", estado: "Sin suscripción". Anotar `userId`.
2. Clic en "Suscribirse" (o ir al Paywall vía estado `authenticated-no-entitlement` forzado).
3. Elegir plan "Overlays" → clic "Suscribirse".
4. Expected: navegador externo abre `https://checkout.stripe.com/c/pay/...`.
5. Completar con tarjeta `4242 4242 4242 4242`, fecha futura, cualquier CVC.
6. Expected: Stripe redirige al servidor local `http://127.0.0.1:39261/checkout/callback` (decisión B), que avisa a la app para revalidar.

- [ ] **Step 3: Verificar entitlements**

1. Volver a la app (alt-tab o la app se reenfoca).
2. Expected: `LicenseBanner` desaparece o cambia a "Activo". Topbar desbloquea secciones Pro.
3. Ir a Ajustes → Cuenta. Expected: Plan "Overlays", estado "Activo", entitlement `overlays` listado.
4. Widget Studio → seleccionar Relative (widget Pro). Expected: no bloqueado, accesible.

- [ ] **Step 4: Verificar en Supabase**

```sql
select * from public.user_entitlements where user_id = '<e2e-test uuid>';
select * from public.stripe_customers where user_id = '<e2e-test uuid>';
select * from public.stripe_subscriptions where user_id = '<e2e-test uuid>';
```

Expected: 1 fila de cada con el entitlement `overlays`, customer y subscription activos.
