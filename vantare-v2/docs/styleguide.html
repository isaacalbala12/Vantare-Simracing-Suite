Do / Don'tReglas críticas para mantener consistencia

✓ Hacer
text-white/60 para texto secundario.
✗ Evitar
text-gray-400 para texto secundario (Tailwind gray).
✓ Hacer
Acento rojo: bg-vantare-red-500/15 text-vantare-red-400.
✗ Evitar
Acento naranja #FF6B35 en producción (no es la marca).
✓ Hacer
Chrome UI en uppercase tracking-widest text-xs font-bold.
✗ Evitar
Chrome UI en text-base font-normal sin tracking.
✓ Hacer
Datos numéricos en font-mono font-black tracking-tight.
✗ Evitar
Datos numéricos con font-tech (no existe) o sans-serif.
✓ Hacer
Botón primario con gradiente from-vantare-red-500 to-[#9a0606].
✗ Evitar
Botón primario con bg-blue-500 o cualquier color fuera de la paleta.
✓ Hacer
Glass via bg-white/[0.02] border-white/10 backdrop-blur-sm.
✗ Evitar
backdrop-blur global en body (consume FPS en OBS).

Botón primario

<!-- Reutilizar la clase utilitaria .btn-primary definida en index.css -->
<button class="btn-primary px-4 py-2 rounded-lg font-bold uppercase tracking-widest text-xs">
  Acción principal
</button>

<!-- O con Tailwind puro: -->
<button class="px-4 py-2 rounded-lg font-bold uppercase tracking-widest text-xs
  bg-gradient-to-br from-vantare-red-500 to-[#9a0606]
  border border-white/10 shadow-vantare-red-900/20
  hover:from-vantare-red-400 hover:to-vantare-red-600
  transition-colors">
  Acción principal
</button>
