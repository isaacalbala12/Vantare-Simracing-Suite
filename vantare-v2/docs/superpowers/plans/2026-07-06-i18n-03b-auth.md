# I18N-03b — Traducción de Auth, Paywall y AccountSettings

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrar la copy hardcodeada en español de `LoginScreen`, `PaywallScreen`, `AccountSettings`, `LicenseBanner` y `UnconfiguredScreen` al sistema i18n (`t()`) con paridad es/en/pt/it.

**Architecture:** Añadir keys nuevas a los 4 diccionarios (ya existen `frontend/src/i18n/locales/{es,en,pt,it}.ts`), reemplazar strings hardcodeados por `t("key")` en cada componente usando `useI18n()` (ya disponible gracias al provider global de `I18N-02`), y añadir tests de paridad que verifiquen que todas las keys de auth existen en los 4 idiomas.
> **⚠️ NOTA DE ORDEN (decisión I, 2026-07-06):** I18N-03b se ejecuta en **F1, ANTES que CHECKOUT-01 (F2)**. I18N-03b traduce los strings de `PaywallScreen` (`paywall.*`) a `t()`. CHECKOUT-01 reescribe `handleSubscribe` pero DEBE respetar esos `t()` ya existentes (no volver a hardcodear español). Si por error CHECKOUT-01 corre primero, el agente de I18N-03b debe migrar también los nuevos strings que CHECKOUT-01 introduzca.
> Las traducciones `pt`/`it` se completan por el agente ejecutor (el plan ya deja el bloque en `...`); la fuente de verdad es `es.ts`.

**Tech Stack:** React 19, TypeScript, i18n interno (`frontend/src/i18n/`), Vitest.

---

## Contexto

- El provider global `I18nProvider` ya está montado en `HubApp` (I18N-02); `useI18n()` → `t()` funciona en cualquier pantalla del Hub sin montar provider local.
- Los diccionarios actuales (`es.ts`, `en.ts`, `pt.ts`, `it.ts`) tienen keys de `nav.*`, `widget.*` y `onboarding.*` pero NADA de `auth.*`, `paywall.*`, `account.*`, `license.*`.
- El test de paridad `i18n.test.ts` ya verifica que todas las keys de `DEFAULT_LOCALE` existen en el resto de locales.
- Este plan solo toca copy visible de auth; no traduce Widget Studio, dashboard, calendar ni launcher (cada feature tiene su corte i18n en `I18N-03`).
