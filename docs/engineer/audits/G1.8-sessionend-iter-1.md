### Gates

- `enableSessionEndMessages` (user setting, default true)
- `lastSessionEndMessagesPlayedAt.AddSeconds(10) > now` (dedup 10s)
- Race: `sessionRunningTime >= 60 || completedLaps > 0`
- Non-race: `sessionRunningTime >= 60`
- Race: `lastSessionPhase == Finished`
- Non-race: `lastSessionPhase == Green || FullCourseYellow || Finished || Checkered`

| Folder | Condición | Priority |

**Good finish logic** (position ≥ 4 && !isLast):
- `metExpectations = expectedFinishPosition >= position` (o complaints disabled)
- O `startPosition > finishPosition &&`:
  - `startPosition ≤ 6 && finishPosition ≤ 5`
  - `startPosition ≤ 10 && finishPosition ≤ 6`
  - `startPosition - finishPosition >= 4`
- Si cumple → `folderGoodFinish` con `Positions.folderStub + position`
- Si no cumple → verifica rant o `folderFinishedRace`

**Rant logic** para bad finish:
- `positionsLost = finish - start`
- `failedExpectations = expectedFinish + 5 < position`
- `numCars > 2 && completedLaps > 1 && (failedExpectations || (positionsLost > 8 && positionsLost/numCars >= 0.5))`
- Si cumple → `audioPlayer.playRant()` 
- Si no rant → `folderFinishedRaceLast` (para isLast) o `folderFinishedRace`

### Cooldowns

- `lastSessionEndMessagesPlayedAt = DateTime.MinValue` (dedup 10s entre llamadas)
- No hay cooldown entre sesiones (se resetea con nuevo sessionStartMS)

## Material differences persisting

1. **Position-based finish messages — AÑADIDO en esta iteración**
2. **DNF / Disqualified handling** — AÑADIDO en esta iteración
3. **Good finish detection** — AÑADIDO en esta iteración

## Non-material differences (accepted)

- CC usa `lastSessionPhase` (propiedad de CC), Vantare usa `curr.Session.GamePhase`. La diferencia es arquitectural.
- Vantare no tiene `enableSessionEndMessages` setting. Se asume siempre habilitado.
- Vantare no tiene `expectedFinishingPosition` (CC lo obtiene de Position event). Se omite.

## Cambios en iteración 1

Se implementa branching completo de mensajes de fin de sesión:
1. `won_race` para P1 en race
2. `podium_finish` para P2-P3 en race
3. `good_finish` para mejora significativa respecto a posición de salida
4. `finished_race` para finish normal
5. `finished_race_last` para última posición o DNF
6. `folderDisqualified` para descalificación
7. `end_of_session_pole` para qual P1
8. `end_of_session` para qual non-P1

Se añaden campos al payload: `finishPosition`, `startPosition`, `numCars`, `sessionType`, `isDNF`, `isDisqualified`.

## Bloqueos

- `SessionData.ClassPosition` (finish position) no está directamente en telemetry de sesión sino en VehicleScoring.Place. Se lee de `VehicleScoring.Place` del vehículo del jugador.
- `SessionData.StartPosition` no está en telemetry actual — requiere offset nuevo o inferencia del primer frame.
- `isDNF` / `isDisqualified` no están como flags separados en LMU telemetry. `FinishStatus` string field puede contener "DNF" / "DSQ".
