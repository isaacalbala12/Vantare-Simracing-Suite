### Mensajes disparados

Niveles de combustible (absolutos):
- `fuel/one_litre_remaining` — priority 10, cuando fuel <= 1L
- `fuel/two_laps_fuel` — (2L warning en modo métrico, priority variable)
- `fuel/one_gallon_remaining` — priority 10, cuando fuel <= 1 galón (~3.785L)
- `fuel/half_a_gallon_remaining` — priority 10, cuando fuel <= 0.5 galón
- `fuel/half_tank_warning` — priority 0, cuando fuel <= 50% del tanque
- `fuel/plenty_of_fuel` — priority variable

Estimaciones por vueltas (en carreras de número fijo de vueltas):
- `fuel/four_laps_fuel` — priority 3, cuando estimatedLapsLeft == 4
- `fuel/three_laps_fuel` — priority 5, cuando estimatedLapsLeft == 3
- `fuel/two_laps_fuel` — priority 7, cuando estimatedLapsLeft == 2
- `fuel/one_lap_fuel` — priority 10, cuando estimatedLapsLeft == 1
- `mandatory_pit_stops/pit_this_lap` — priority 7, cuando 1 lap left + sector3

Estimaciones por tiempo (en carreras cronometradas):
- `fuel/ten_minutes_fuel` — priority 3, ~10 min remaining
- `fuel/five_minutes_fuel` — priority 7, ~5 min remaining
- `fuel/two_minutes_fuel` — priority 10, ~2 min remaining
- `fuel/half_distance_good_fuel` — priority 5 (mitad de carrera, suficiente)
- `fuel/half_distance_low_fuel` — priority 7 (mitad de carrera, insuficiente)
- `fuel/about_to_run_out` + `pit_this_lap` — priority 10

### Gates

- `fuelUseActive` — solo procesar si el uso de combustible está activo
- `!GameStateData.onManualFormationLap` — vuelta de formación
- `SessionRunningTime > 15` — primeros 15s de sesión
- `CarSpeed > 10` — en práctica/clasificación
- `!onLowFuelRun` — no emitir mensajes en low fuel run (prac/qual)
- `canPlayFuelMessage()` — cooldown 30s desde `lastFuelCall`
- `!currentGameState.PitData.InPitlane` — no en pits
- `currentGameState.FuelData.FuelLeft > 0` — combustible positivo
- `fuelLevelWindowByTime.Count > 0` — datos disponibles
- `lapsCompletedSinceFuelReset > 0` — al menos una vuelza completada

### Cooldowns

- `lastFuelCall` — mínimo 30s entre mensajes automáticos de combustible (canPlayFuelMessage)
- `nextFuelStatusCheck` — cada 5s para comprobaciones de estado
- `nextFuelPitWindowOpenCheck` — cada 5s para comprobaciones de ventana de pits
- `fuelUseSampleTime = 60s` — intervalo entre muestras de ventana por tiempo
- `gameTimeAtLastFuelWindowUpdate` — control de frecuencia de muestreo

## Diferencias materiales que persisten

1. **Sin 2L warning** — CC anuncia "2 litres remaining" además de "1 litre". Vantare solo anuncia 1L. Impacto: el piloto no recibe advertencia intermedia.
2. **Sin `canPlayFuelMessage` cooldown** — CC impone 30s entre mensajes automáticos de combustible. Vantare emite sin restricción. Impacto: posible spam de mensajes en situaciones límite.
3. **Sin re-arm en CC (one-shot)** — CC no rearma las advertencias 1L/half-tank una vez disparadas. Vantare sí rearma (con histéresis). Impacto: Vantare puede advertir múltiples veces si el nivel oscila cerca del umbral. El comportamiento de CC es más conservador.
4. **Sin per-lap consumption** — CC calcula consumo por vuelta basado en ventana deslizante. Vantare no tiene este cálculo, necesario para estimaciones de vueltas restantes.
5. **Sin per-minute consumption** — CC calcula consumo por minuto para carreras cronometradas. Vantare no lo tiene.
6. **Sin per-lap remaining estimates** — CC anuncia cuando quedan 4, 3, 2, 1 vueltas de combustible. Vantare no da ninguna estimación de autonomía.
7. **Sin per-minute remaining** — CC anuncia 10, 5, 2 minutos de combustible restante. Vantare no da estimación temporal.
8. **Sin half-distance estimate** — CC evalúa combustible en la mitad de la carrera y anuncia "good fuel" / "low fuel". Vantare no lo hace.
9. **Sin "pit for fuel now"** — CC combina "about to run out" con "pit this lap" cuando el combustible es crítico. Vantare no tiene este mensaje.

## Diferencias NO materiales (aceptadas)

- Vantare usa `EventLowFuel1Litre` / `EventLowFuelHalfTank` como text keys vs CC `fuel/one_litre_remaining` / `fuel/half_tank_warning` — diferencia de naming en pipeline
- Vantare no tiene soporte de galones (imperial). Se asume métrico en alpha 1.
- Vantare no tiene voice response support — fuera de alcance alpha 1.
- Vantare no tiene fuel window prediction — requiere consumo por vuelta, que es G2.x.

## Plan para próxima iteración (iter-2)

1. **Añadir 2L warning**: cuando `fuel < 2.0` y no se ha disparado aún. Usar flag `played2LitreWarning`.
2. **Añadir `canPlayFuelMessage` gate**: mantener `lastFuelCall` timestamp, permitir mensajes solo si han pasado 30s desde el último.
3. **Cambiar re-arm a one-shot (CC behavior)**: eliminar histéresis de re-arm para 1L y half-tank. Una vez disparados, no se repiten. Esto alinea con CC.
4. **Añadir per-lap consumption básico**: calcular consumo medio en ventana de N vueltas cuando el frame tiene `Fuel` y `LapDistance`.
5. **Añadir half-distance fuel estimate**: detectar cuando `CompletedLaps == SessionNumberOfLaps / 2`, calcular combustible necesario vs restante, emitir `fuel.low_half` o `fuel.good_half`.

## Bloqueos

- Per-lap consumption requiere que LMU exponga `CompletedLaps` y `LapDistance` consistentes en el frame. NO_VERIFICADO — probablemente OK pero requiere validación.
- Half-distance estimate requiere `SessionNumberOfLaps` o `SessionTotalRunTime`. Disponible en `frame.Session` si LMU lo expone.
- Unit conversion (galones) requiere user setting `use_metric`. PARITY_BLOCKED hasta que runtime exponga settings.
