## Iteración 3 — Cambios aplicados

1. **`EventLeaderPitted`**:
   - Se dispara cuando un oponente con `Place == 1` (líder) entra a pits.
   - Cooldown por oponente usando `canFire()` existente.
   - Tests: `TestMonitor_LeaderPitted`.

2. **`EventCarAheadPitted`**:
   - Se dispara cuando un oponente con `Place == playerPlace - 1` entra a pits.
   - No se dispara si el jugador es líder (playerPlace == 1).
   - Tests: `TestMonitor_CarAheadPitted`, `TestMonitor_CarAheadNotPittedWhenPlayerIsLeader`.

3. **`EventCarBehindPitted`**:
   - Se dispara cuando un oponente con `Place == playerPlace + 1` entra a pits.
   - Tests: `TestMonitor_CarBehindPitted`.

4. **`EventLeadChanged`**:
   - Se dispara cuando `player.Place == 1` (jugador es ahora líder) y antes no lo era (rising edge).
   - Cooldown 60s, con zero sentinel para primer disparo.
   - Tests: `TestMonitor_LeadChanged`, `TestMonitor_LeadChanged_NoFireIfAlreadyLeader`, `TestMonitor_LeadChanged_Cooldown`.

5. **`minImprovementBeforeReadingOpponentRaceTime = 0.05`**:
   - Nueva constante `minImprovementForBestLap = 0.05`.
   - Best lap solo se dispara si `v.BestLapTime < st.BestLapTime - minImprovementForBestLap`.
   - Tests: `TestMonitor_MinImprovementForBestLap`, `TestMonitor_MinImprovementForBestLap_AllowsLargeImprovement`.

6. **Tests actualizados**:
   - Nuevo helper `mkOpponentsFramePlace` para crear frames con place específico.
   - Test `TestMonitor_OpponentBestLap` ajustado: mejora 105.5→104.2 = 1.3s, supera threshold.

## Re-auditoría post-iter-3

**Diferencias materiales que PERSISTEN:**

1. **Retirements & Disqualifications** — CC detecta `retriedDriverNames`/`disqualifiedDriverNames`. Vantare no lee esto. `PARITY_BLOCKED` — requiere leer scoring buffer extendido.

2. **Lapped car ahead detection** — CC detecta coches con vuelta perdida delante (`folderOneLapAhead/Behind`). G3+ scope.

3. **Practice/Qualifying probabilistic reporting** — CC usa random gate para best laps en practice/qual. Vantare usa `minLaps >= 2` siempre. G3+ scope.

4. **Priority 3 vs 10** — CC usa priority 3 para opponents messages (no urgentes). Vantare usa 10. Aceptado (uniformidad de PriorityNormal).

5. **Car ahead change detection** — CC detecta cuando cambia el coche de delante con cooldown 30s. No implementado. G3+ scope.

6. **Next car is... announcement** — CC anuncia "next car is [name]" con delay. G3+ scope.

7. **Time delta gates** — CC requiere `TimeDeltaFront > 3` para car-ahead-pitted. Vantare no verifica delta. Diferido a iter-4.

**Diferencias NO materiales (aceptadas):**
- CC usa `PitData.LeaderIsPitting` / `CarInFrontIsPitting` / `CarBehindIsPitting` flags calculados; Vantare calcula desde `Place` en tiempo real. Equivalente funcional.
