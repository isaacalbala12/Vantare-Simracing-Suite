# AUDIT-01 — license_events y Discord role sync en la Edge Function

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cerrar los follow-ups de la Edge Function: escribir `license_events` por cada webhook procesado, y emitir un webhook a Discord cuando cambie el tier de un usuario.

**Architecture:** Modificar los handlers de la EF (`stripe-webhook/index.ts`) para que cada evento procesado inserte una fila en `public.license_events` (con el tipo de evento, user_id y payload resumido). Añadir un helper `notifyDiscord` que emita un POST a un webhook de Discord configurado como secreto (aviso al canal de soporte del equipo, NO asignación de rol). Ambos son puramente aditivos; no cambian la lógica de entitlements existente.

**Tech Stack:** Deno/TypeScript (Supabase Edge Functions), Supabase client (service-role), fetch a Discord webhook.

---
> **⚠️ CORRECCIÓN (revisión 2026-07-06, decisión D):**
> - `syncDiscordRole` se renombra a **`notifyDiscord`**. El objetivo NO es asignar roles de Discord (eso requiere un bot aparte, fuera de scope). El código envía un **mensaje a un canal de soporte del equipo** vía webhook de Discord, para que el equipo sepa que un usuario cambió de plan.
> - El webhook de Discord es un **aviso al equipo**, no al usuario.
> - Los tests asumen API del mock que hoy NO existe (`ctx.env`, `ctx.getTableRows()`, y `from("license_events").insert`). El mock de `index.test.ts` debe extenderse antes (ver Task 0). Sin esto, los tests RED fallan por el mock, no por la función.

- [ ] **Step 2: Implementar notifyDiscord**

async function notifyDiscord(
  userId: string,
  email: string,
  productKeys: string[],
  tier: string,
): Promise<void> {
  // NOTA (decisión D): esto es un AVISO al canal de soporte del equipo,
  // no asignación de rol. No requiere bot de Discord.
  const webhookUrl = Deno.env.get("DISCORD_ROLE_SYNC_WEBHOOK_URL");
  if (!webhookUrl) return; // opcional

  try {
    const res = await fetch(webhookUrl, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        content: `💳 Nuevo tier: ${tier} para ${email} (${userId}) — ${productKeys.join(", ")}`,
      }),
    });
    if (!res.ok) {
      console.error("Discord notify failed:", res.status);
    }
  } catch (err) {
    console.error("Discord notify error:", err);
  }
}
```

function classifyTier(keys: string[]): string {
  if (keys.includes("visionary_backer")) return "visionary_backer";
  if (keys.includes("pro_founder")) return "pro_founder";
  if (keys.includes("founder")) return "founder";
  if (keys.includes("supporter")) return "supporter";
  if (keys.includes("beta_access")) return "beta_access";
  const hasOverlays = keys.includes("overlays");
  const hasEngineer = keys.includes("engineer");
  if (hasOverlays && hasEngineer) return "suite";
  if (hasOverlays) return "overlays";
  if (hasEngineer) return "engineer";
  return "free";
}
```
