### Sector-based cadence

Las frecuencias se traducen a espera de sectores:

```
closeAheadMinSectorWait = 11 - gapAheadReportFrequency
closeAheadMaxSectorWait = closeAheadMinSectorWait + 5
gapAheadMinSectorWait = 13 - gapAheadReportFrequency
gapAheadMaxSectorWait = gapAheadMinSectorWait + 5
closeBehindMinSectorWait = 13 - gapBehindReportFrequency
closeBehindMaxSectorWait = closeBehindMinSectorWait + 5
gapBehindMinSectorWait = 13 - gapBehindReportFrequency
gapBehindMaxSectorWait = gapBehindMinSectorWait + 5
```

### Gap Status Detection

**GapStatus.CLOSE** (coche muy cerca):
- 2 sectores consecutivos con gap < 0.5s, o
- 3 sectores con gap < 0.7s, o
- 4 sectores con gap < 0.8s

**GapStatus.INCREASING**:
- 3 muestras consecutivas: gap[0] > gap[1] > gap[2] (creciente)

**GapStatus.DECREASING**:
- 3 muestras consecutivas: gap[0] < gap[1] < gap[2] (decreciente)

**GapStatus.OTHER**:
- Gap no ha cambiado más de 1 segundo en 3 muestras

**Gates para reportar Gaps:**
- `!currentGameState.PitData.InPitlane`
- `enableGapMessages` (bool setting)
- `!currentGameState.FlagData.currentLapIsFCY`
- `!isNearRaceEnd(currentGameState)`
- `!PitStops.isPittingThisLap`
- Gap mínimo: `> 0.5s` y `< 20s`
- gap mínimo para leer: `> 50ms`

### Being Held Up

- Se activa después de 60s detrás del mismo coche (close gap)
- `!GlobalBehaviourSettings.useOvalLogic`
- Cooldown: por sectores (closeAheadMinSectorWait..closeAheadMaxSectorWait)
- Adjunta corner attack suggestion si hay track landmark data

### Being Pressured

- Similar a being held up pero para coche detrás
- Cooldown: closeBehindMinSectorWait..closeBehindMaxSectorWait
- Adjunta corner defend suggestion

## Non-material differences (accepted)

- Vantare no tiene `enableGapMessages` setting. Se asume siempre habilitado en race.
- Vantare no tiene `justTheFacts` setting. No se aplica gate.
- Vantare no trackea `gapPoints` por track. Se omite soporte.
- Vantare no tiene sistema de DelayedMessageEvent (resolución de gap diferida). Se usa valor instantáneo.

## Cambios en iteración 1

Se implementa:
1. Gap status detection con 3-sample history: CLOSE, INCREASING, DECREASING, OTHER, NONE
2. Thresholds: close < 0.5s (2 samples), increasing/decreasing (3 samples monotonic)
3. Gap min = 0.5s, max = 20s (CC parity)
4. Gap readable min = 50ms
5. Pit lane gate: skip if player.InPits
6. FCY gate: skip if yellow flag
7. Evento ampliado: `gapStatusInFront`, `gapStatusBehind`, `gapSecs`
