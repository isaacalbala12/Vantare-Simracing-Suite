### Gates

- `checkPushToGain = sessionRunningTime - lastPositionFrontChange < 60s`
- `checkPushToHold = sessionRunningTime - lastPositionBehindChange < 60s`
- `!GlobalBehaviourSettings.justTheFacts`
- `!playedNearEndTimePush` (para time-based)
- `!playedNearEndLapsPush` (para laps-based)

### checkGaps logic

- Si `checkPushToGain && ClassPosition > 1`:
  - Obtiene best lap del oponente de enfrente (últimos 4 laps)
  - Si `(opponentBest - playerBest) * lapsLeft > timeDeltaFront` → alcanzable
    - `ClassPosition == 2` → `push_to_get_win` (priority 5)
    - `ClassPosition == 3` → `push_to_get_second` (priority 5)
    - `ClassPosition == 4` → `push_to_get_third` (priority 5)
    - else → `push_to_improve` (priority 5)
- Si `checkPushToHold && !isLast`:
  - Obtiene best lap del oponente detrás (últimos 4 laps)
  - Si `(playerBest - opponentBest) * lapsLeft > timeDeltaBehind` → nos van a coger
    - `push_to_hold_position` (priority 3)

## Non-material differences (accepted)

- Vantare no tiene `justTheFacts` (CC setting global). Se omite gate.
- Vantare no tiene `pearlsOfWisdom`. No se añaden pearls a mensajes.
- Vantare no implementa brake/tyre temp warnings en pit exit. `PARITY_BLOCKED` (LMU no expone brake temp por separado en telemetry básica).

## Cambios en iteración 1

Se implementa:
1. Track length class inferido de `Session.TrackLength`
2. Push windows por laps (MEDIUM≤4, LONG≤2, VERY_LONG≤1) y por tiempo (120s < remaining < 240s)
3. `minTimeToBeInThisPosition = 60s` gate
4. `checkPushToGain` (position change ahead < 60s) y `checkPushToHold` (position change behind < 60s)
5. Mensajes: push_to_get_win, push_to_get_second, push_to_get_third, push_to_improve, push_to_hold_position
6. `lapsToCountBackForOpponentBest = 4` para cálculo de best lap de oponente

## Bloqueos

- `TrackLengthClass` no está disponible como enum en Vantare telemetry. Se infiere de `Session.TrackLength` usando thresholds CC:
  - > 20000m = VERY_LONG, > 10000m = LONG, < 2400m = SHORT, < 1000m = VERY_SHORT, else MEDIUM
- `SessionData.TimeDeltaFront/Behind` no está directamente en telemetry actual — se usa `TimeBehindNext` y `TimeBehindLeader`.
- `SessionData.GameTimeAtLastPositionFrontChange/BehindChange` no disponible — se añade `PositionLastChangeTime` al frame.
- `SessionData.PlayerLapTimeSessionBest` disponible como `BestLapTime`.
- `OpponentData.OpponentLapData` (últimos N laps) no disponible en Vantare — necesario para cálculo de best lap de oponente en ventana.
